## 1. システム概要

### 1-1. 何ができるシステムか

**ほいくシフト**は、保育園のシフト管理を画面上で簡単に行うためのWebアプリケーションです。

| こんな課題を解決します | |
|---|---|
| Excelや紙でのシフト作成が毎月大変 | → 画面上でクリックするだけでシフトが入力できる |
| 職員配置基準を満たしているか毎回確認が必要 | → 不足日を自動で赤くハイライト表示 |
| 希望休の集約と反映に時間がかかる | → 職員がスマホから申請 → 管理者がワンタップで承認 |
| 勤務時間の集計が手間 | → 月次の合計時間を自動計算 |

### 1-2. システムの特徴

- **インターネット接続のみで動作** — インストール不要、ブラウザで開くだけ
- **データベース不要** — すべてのデータはブラウザ内に保存（localStorage）
- **スマートフォン対応** — スマホからでも快適に操作できるよう設計
- **シンプルな画面** — システムが苦手な方でも直感的に操作できる

## 4. 管理者向け使用マニュアル

### 4-1. 初期設定（最初に一度だけ行う作業）

#### STEP 1: 保育園の基本情報を設定する

1. 左メニュー（PC）またはボトムナビ（スマホ）の **「設定」** をタップ
2. 「🏫 保育園情報」欄を入力する

| 項目 | 説明 | 例 |
|---|---|---|
| 園名 | 保育園の名前 | ひまわり保育園 |
| 開園時間 | 最も早い開園時間 | 07:00 |
| 閉園時間 | 最も遅い閉園時間 | 20:00 |
| 希望休申請の締め切り | 毎月何日まで申請を受け付けるか | 15（毎月15日締め） |

3. 「保存する」ボタンをタップ

---

#### STEP 2: クラスを設定する

1. 設定画面の「👶 クラス設定」欄を開く
2. **「追加」** ボタンから各クラスを登録する

| 項目 | 説明 |
|---|---|
| クラス名 | 例: 0歳児クラス、ひよこ組 など |
| 年齢 | 0〜5歳から選択 |
| 児童数 | 現在の在籍児童数 |

> **必要保育士数は自動計算されます。**  
> 例: 0歳児が6名 → 最低2名の保育士が必要（3:1の基準）

**配置基準（2024〜2025年度改正版）**

| 年齢 | 子ども : 保育士 |
|---|---|
| 0歳児 | 3 : 1 |
| 1歳児 | 6 : 1 |
| 2歳児 | 6 : 1 |
| 3歳児 | 15 : 1（2024年改正） |
| 4〜5歳児 | 25 : 1（2024年改正） |

---

#### STEP 3: シフトパターンを確認・カスタマイズする

1. 設定画面の「🕐 シフトパターン」欄を開く
2. 初期値として以下の8パターンが登録されています

| パターン名 | 時間帯 | 色 |
|---|---|---|
| 早番 | 07:00〜16:00（実働8h） | 水色 |
| 中番 | 09:00〜18:00（実働8h） | 緑 |
| 遅番 | 11:00〜20:00（実働8h） | オレンジ |
| 通常 | 08:30〜17:30（実働8h） | グレー |
| 午前P | 08:30〜12:30（実働3.5h） | 紫 |
| 午後P | 13:30〜17:30（実働3.5h） | ピンク |
| 休み | — | 赤 |
| 有給 | — | 紫 |

> 「追加」ボタンで独自のパターンを作成できます。  
> 「休み扱い」にチェックを入れると労働時間が0になります。

---

#### STEP 4: 職員を登録する

1. 左メニューの **「職員管理」** をタップ
2. **「追加」** ボタンをタップ
3. 以下の項目を入力して「追加する」をタップ

| 項目 | 必須 | 説明 |
|---|---|---|
| 氏名 | ◎ | フルネームで入力（例: 田中 花子） |
| ロール | ◎ | 管理者 or 職員 |
| 雇用形態 | ◎ | 正社員 or パート |
| 週所定時間 | ◎ | 例: 正社員は40、パートは20 |
| メモ | — | 例: 火・木のみ勤務 |
| カラー | ◎ | シフト表での識別色 |

---

### 4-2. 毎月のシフト作成手順

#### STEP 1: 希望休の申請期限を確認する

- 職員からの希望休申請は「ホーム」画面のアラートに表示されます
- 申請一覧で全件確認し、承認または却下を行ってください
- **申請一覧の操作方法:**
  1. 「申請一覧」をタップ
  2. 「審査中」タブで未処理の申請を確認
  3. 各申請の「承認する」または「却下する」ボタンをタップ

---

#### STEP 2: シフト表を開く

1. 「シフト表」をタップ
2. 画面上部の `◀ ▶` ボタンで対象月に移動

---

#### STEP 3: シフトを入力する

1. シフトを入力したい **職員名の行 × 日付の列のセルをタップ**
2. 下から「シフト選択」シートが表示される
3. パターン（早番・中番・遅番など）をタップして選択
4. シフトが即時反映される

> シフトを空白に戻したい場合は「シフトをクリア（空白にする）」ボタンをタップします。

---

#### STEP 4: 配置基準を確認する

- シフト表の上部に配置基準の充足状況が表示されます
- **緑の表示** → 全日で基準を満たしている
- **黄色の表示** → 不足している日がある → 日付欄の「▲」マークで確認

不足している日は職員を追加するかシフトパターンを変更してください。

---

#### STEP 5: 職員の勤務時間を確認する

- シフト表の各行右端に **月間合計勤務時間** が表示されます
- ホーム画面の「今月の勤務時間」バーグラフで全員の状況を一覧確認できます
- 週平均40時間を超えると赤色でアラートが表示されます

> **労働時間の計算ルール**  
> 各シフトパターンの勤務時間から**休憩1時間を差し引いた実働時間**で計算します。  
> 例: 早番（07:00〜16:00）= 9時間 − 休憩1時間 = **実働8時間**

---

### 4-3. 職員情報の編集・削除

#### 編集する

1. 「職員管理」をタップ
2. 編集したい職員のカードの ✏️（鉛筆）アイコンをタップ
3. 情報を変更して「変更を保存する」をタップ

#### 削除する

1. 「職員管理」をタップ
2. 削除したい職員のカードの 🗑️（ゴミ箱）アイコンをタップ
3. 確認ダイアログで「削除する」をタップ

> **注意:** 削除したデータは元に戻せません。

## 5. 職員向け使用マニュアル

### 5-1. 自分のシフトを確認する

1. ボトムナビ（画面下）の **「シフト確認」** をタップ
2. 月次のシフト表が表示されます
3. 自分の行を横スクロールして確認してください

> 月の切り替えは画面上部の `◀ ▶` ボタンで行います。

---

### 5-2. 希望休・有給を申請する

1. ボトムナビの **「希望休申請」** をタップ
2. 右上の **「申請する」** ボタンをタップ
3. 以下を入力して「申請を送る 🌸」をタップ

| 項目 | 説明 |
|---|---|
| 申請する日 | カレンダーから日付を選択（本日以降のみ選択可） |
| 休みの種類 | 🙏 希望休 または 📝 有給休暇 |
| 理由（任意） | 例: 通院のため |

---

### 5-3. 申請の状況を確認する

- 申請後は「希望休申請」画面の一覧に表示されます
- ステータスは以下の3種類です

| ステータス | 意味 |
|---|---|
| ⏳ 審査中 | 管理者が確認中。しばらくお待ちください |
| ✅ 承認済 | 申請が承認されました |
| ❌ 却下 | 申請が却下されました。管理者に確認してください |

---

### 5-4. 表示する職員を切り替える（複数の職員がいる場合）

- 画面右上のボタン（🌸 名前）をタップ
- ドロップダウンから自分の名前を選択する

## 6. データの保存について

### 6-1. 保存の仕組み

本システムはデータベースを使用せず、**ブラウザの「localStorage」**にデータを保存します。

| 特徴 | 詳細 |
|---|---|
| 自動保存 | 操作するたびに自動でブラウザ内に保存される |
| ブラウザを閉じても保持 | 次に開いたときも同じデータが表示される |
| サーバー不要 | インターネット接続があれば動作する |

> **保存先キー名:** `hoiku-shift-store`

---

### 6-2. データが消えるケース（重要）

以下の操作を行うとデータが**すべて消えます**。必ず事前にご確認ください。

| 操作 | データへの影響 |
|---|---|
| ブラウザの「サイトデータを削除」 | ⚠️ 消える |
| ブラウザのキャッシュクリア（全消去） | ⚠️ 消える |
| シークレット（プライベート）ウィンドウで使用 | ⚠️ ウィンドウを閉じると消える |
| 別のブラウザで開く | ⚠️ 別のブラウザには引き継がれない |
| 別のデバイスで開く | ⚠️ データは共有されない |

---

### 6-3. 複数人での使用について

現在のバージョンは**1台のPC・端末で使う**ことを前提としています。  
複数の担当者や複数の端末で同じデータを共有したい場合は、**Firebase連携版（将来対応予定）**をご利用ください。

---

### 6-4. データのバックアップ方法（手動）

ブラウザの開発者ツールを使って手動でデータをコピーできます。

```
1. Chrome でシステムを開く
2. F12 キー（または右クリック → 「検証」）を押す
3. 「Application」タブ → 「Local Storage」 → サイトのURLを選択
4. 「hoiku-shift-store」の値をコピーしてテキストファイルに貼り付けて保存
```

## 7. 法的・制度的なルール

本システムはシフト作成の補助ツールです。以下のルールは**管理者が必ず確認**してください。

### 7-1. 労働基準法の主なルール

| ルール | 内容 |
|---|---|
| 1日の労働時間 | 8時間以内（休憩1時間以上含む） |
| 週の労働時間 | 40時間以内 |
| 残業 | 36協定の締結が必要。月45時間超は要注意 |
| 有給休暇 | 年10日以上付与の職員は年5日取得が義務 |
| インターバル | 勤務終了から次の勤務開始まで十分な休息時間を確保 |

> システム上でも週平均40時間超の職員にはアラートが表示されます。

---

### 7-2. 保育士配置基準（2024〜2025年度改正版）

国が定める「**その時間帯に実際にいる保育士の人数**」での基準です。

| 年齢 | 子ども : 保育士 | 備考 |
|---|---|---|
| 0歳児 | 3 : 1 | |
| 1歳児 | 6 : 1 | 2025年度より加算措置で5:1も可能 |
| 2歳児 | 6 : 1 | |
| 3歳児 | 15 : 1 | 2024年改正（旧20:1） |
| 4〜5歳児 | 25 : 1 | 2024年改正（旧30:1） |

> 本システムはクラス別の在籍児童数から必要保育士数を自動計算し、不足日をシフト表上で警告します。

---

### 7-3. 確認が特に必要なシフトパターン

| 要注意パターン | 内容 |
|---|---|
| 遅番の翌日に早番 | 勤務終了〜次の勤務開始の時間が短くなるため注意 |
| 3日連続以上の早番・遅番 | 生活リズムへの影響が大きい |
| パートスタッフへの長時間配置 | 週所定時間を超えないように確認 |

### 8-2. データ構造

#### 職員（Staff）

```typescript
{
  id: string            // 一意のID（例: s_1234567890）
  name: string          // 氏名
  role: 'admin' | 'staff'
  employment: 'fulltime' | 'parttime'
  weeklyHours: number   // 週所定時間（h）
  color: string         // シフト表の識別色（HEX）
  note: string          // メモ
}
```

#### シフトパターン（ShiftPattern）

```typescript
{
  id: string            // 一意のID
  name: string          // パターン名（例: 早番）
  startTime: string     // 開始時間（HH:MM）
  endTime: string       // 終了時間（HH:MM）
  color: string         // テキスト色（HEX）
  bgColor: string       // 背景色（HEX）
  isOff: boolean        // 休み扱いの場合 true
}
```

#### シフトデータ（ShiftData）

```typescript
// shifts[年月][職員ID][日] = { patternId, note }
// 例: shifts["2026-05"]["s1"]["15"] = { patternId: "early", note: "" }
{
  [yearMonth: string]: {
    [staffId: string]: {
      [day: string]: {
        patternId: string
        note: string
      }
    }
  }
}
```

#### 希望休申請（LeaveRequest）

```typescript
{
  id: string            // 一意のID（例: req_1234567890）
  staffId: string       // 申請した職員のID
  date: string          // 申請日（YYYY-MM-DD）
  type: 'dayoff' | 'paid'  // 希望休 or 有給
  reason: string        // 理由（任意）
  status: 'pending' | 'approved' | 'rejected'
  createdAt: string     // 申請日時（ISO文字列）
}
```

---

### 8-3. 労働時間の計算ロジック

```
実働時間 = (終了時間 - 開始時間) - 休憩1時間
例: 早番 07:00〜16:00 = 9時間 - 1時間 = 8時間

月次合計 = その月に入力されたすべてのシフトの実働時間の合計
週平均 = 月次合計 ÷ 4
```

`isOff: true` のパターン（休み・有給）は実働時間 **0h** として計算されます。

### 8-5. デプロイ設定（Netlify）

`netlify.toml` の内容:

```toml
[build]
  command = "npm run build"
  publish = "dist"

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200
```

> `[[redirects]]` はReact Routerのブラウザ履歴ルーティングに必要です。この設定がないとページ更新時に404エラーになります。

## 9. よくある質問（FAQ）

### Q1. データがなくなりました

**A.** ブラウザのキャッシュやサイトデータが削除された可能性があります。  
本バージョンはlocalStorageに保存しているため、ブラウザのデータ削除でデータが消えます。  
重要なデータは「6-4. データのバックアップ方法」を参考に定期的にバックアップしてください。

---

### Q2. スマホで見ると表示が崩れる

**A.** 対応ブラウザ（Safari / Chrome）の最新版をお使いください。  
また、ブラウザの「テキストサイズの自動調整」をオフにすると改善される場合があります。

---

### Q3. シフト表のセルが小さくて押しにくい

**A.** シフト表は横スクロールが必要なため、スマホでは若干操作しにくい場合があります。  
- PCやタブレットでの操作を推奨します
- スマホの場合は画面を横向き（横持ち）にすると見やすくなります

---

### Q4. 配置基準の⚠️マークが消えない

**A.** その日のシフトに出勤している人数が、配置基準の必要人数を下回っています。  
「設定 → クラス設定」の児童数が最新かどうか確認し、シフトに出勤者を追加してください。

---

### Q5. 複数のPCで同じデータを使いたい

**A.** 現在のバージョンでは対応していません。  
データはブラウザ内にのみ保存されるため、別のPCやブラウザには同期されません。  
**将来的にFirebase連携版でデータの共有に対応する予定です。**

---

### Q6. パターンを追加・削除してもシフト表に反映されない

**A.** 削除したパターンが既存のシフトに使用されている場合、そのセルは空白（—）で表示されます。  
削除前に、そのパターンが使用されている月のシフトを別のパターンに変更してください。

---

### Q7. 希望休を申請したが管理者に届いているか不安

**A.** 申請が正常に送られると、申請一覧に「⏳ 審査中」として表示されます。  
管理者のホーム画面にも「未処理の申請が〇件あります」というアラートが表示されます。
