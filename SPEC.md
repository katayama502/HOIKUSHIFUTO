# 保育園シフト管理・作成システム 仕様書

**プロジェクト名:** HOIKUSHIFUTO  
**バージョン:** 1.0  
**作成日:** 2026-04-26  
**デプロイ先:** Netlify  
**DB:** Firebase (Firestore + Authentication)

---

## 1. システム概要

保育園における職員のシフト作成・管理を効率化するWebアプリケーション。  
主な課題である「手作業によるシフト作成の負担」「職員配置基準の確認漏れ」「希望休の集約・反映」を解決する。

---

## 2. 背景・要件定義

### 2-1. 法的制約

| 種別 | 内容 |
|------|------|
| 労働基準法 | 1日8時間以内、週40時間以内 |
| 残業 | 36協定締結が必要。月45時間超は要注意 |
| インターバル | 「遅番翌日の早番」など連続勤務を避ける必要あり |
| 有給休暇 | 年10日以上付与対象者への年5日取得義務 |

### 2-2. 職員配置基準（2024〜2025年度改正版）

| 年齢 | 子ども人数 : 保育士数 |
|------|----------------------|
| 0歳児 | 3:1 |
| 1歳児 | 5:1（2025年度加算対応） / 6:1（法定） |
| 2歳児 | 6:1 |
| 3歳児 | 15:1（2024年度改正。旧20:1） |
| 4〜5歳児 | 25:1（2024年度改正。旧30:1） |

> 各時間帯に「その時間帯にいる実人数」で基準を満たす必要あり（常勤換算不可）

### 2-3. 標準シフトパターン

| 名称 | 時間帯例 | 説明 |
|------|----------|------|
| 早番 | 07:00〜16:00 | 開園〜午前保育 |
| 中番 | 09:00〜18:00 | コアタイム帯 |
| 遅番 | 11:00〜20:00 | 午後〜閉園 |
| 通常 | 08:30〜17:30 | 標準8時間勤務 |
| 半日AM | 08:30〜12:30 | パート短時間 |
| 半日PM | 13:30〜17:30 | パート短時間 |
| 休み | — | 休日・有給など |

> シフトパターンは管理者が自由に追加・編集可能

## 3. ユーザーロール

| ロール | 説明 | 主な操作 |
|--------|------|----------|
| **管理者** (園長・主任) | シフト作成・承認権限あり | シフト作成・編集・確定・配置基準確認 |
| **職員** (保育士・パート) | 自分の予定のみ操作可 | 希望休申請・シフト確認・交代申請 |

## 4. 機能一覧

### 4-1. 認証・ユーザー管理

- Firebase Authentication によるメール/パスワード認証
- ロール管理（管理者 / 職員）
- 職員プロフィール設定
  - 氏名、雇用形態（正社員/パート/派遣）
  - 担当クラス（0〜5歳クラス）
  - 週所定労働時間
  - 資格保有（保育士/幼稚園教諭など）

### 4-2. シフト管理（管理者）

#### シフト作成
- 月単位でのシフト作成画面（カレンダービュー）
- 職員ごとに各日のシフトパターンを割り当て
- ドラッグ&ドロップ対応のシフト編集
- 希望休の反映（希望休申請が自動表示）
- 労働時間の自動集計（週・月単位）
- 法定外労働時間のアラート表示

#### 配置基準チェック
- 各時間帯ごとの配置人数を自動計算・表示
- 配置基準を下回る時間帯を赤色ハイライト表示
- 配置基準充足率の確認パネル

#### シフト確定・通知
- ドラフト / 確定 の2ステータス管理
- 確定時に全職員へ通知（アプリ内通知）
- 確定後の変更は「変更履歴」に記録

#### シフト一覧表示
- 月次カレンダービュー（全員の俯瞰表示）
- 週次ビュー
- 職員別ビュー
- CSV出力

### 4-3. 希望休・申請管理（職員）

- 希望休申請（月次締め切り設定あり）
- 有給休暇申請
- シフト交代申請（相手職員の承認フロー付き）
- 申請一覧・ステータス確認

### 4-4. 勤怠サマリー（管理者）

- 月次の労働時間集計テーブル
- 残業時間・有給消化日数の一覧
- アラート対象職員のハイライト（36協定超え、有給未消化など）

### 4-5. 設定

- 保育園プロフィール設定（名称、開園時間など）
- シフトパターンのカスタマイズ（追加/編集/削除）
- クラス設定（在籍児童数の登録）
- 希望休申請の締め切り日設定

## 7. データ設計（Firestore）

### コレクション構成

```
/organizations/{orgId}
  - name: string
  - openTime: string ("07:00")
  - closeTime: string ("20:00")

/organizations/{orgId}/staff/{staffId}
  - name: string
  - email: string
  - role: "admin" | "staff"
  - employment: "fulltime" | "parttime" | "temp"
  - weeklyHours: number
  - qualifications: string[]
  - assignedClass: string[]
  - uid: string (Firebase Auth UID)

/organizations/{orgId}/classes/{classId}
  - name: string (例: "0歳児クラス")
  - ageGroup: 0 | 1 | 2 | 3 | 4 | 5
  - childrenCount: number
  - requiredStaff: number (自動計算)

/organizations/{orgId}/shiftPatterns/{patternId}
  - name: string (例: "早番")
  - startTime: string ("07:00")
  - endTime: string ("16:00")
  - color: string ("#BAE6FD")
  - workHours: number (自動計算)

/organizations/{orgId}/shifts/{yearMonth}  // 例: "2026-05"
  - status: "draft" | "published"
  - createdBy: string
  - updatedAt: timestamp
  - entries: {
      [staffId]: {
        [date]: {  // "1"〜"31"
          patternId: string
          note: string
        }
      }
    }

/organizations/{orgId}/requests/{requestId}
  - staffId: string
  - type: "dayoff" | "paid" | "swap"
  - targetDate: timestamp
  - status: "pending" | "approved" | "rejected"
  - note: string
  - createdAt: timestamp
  - swapWithStaffId?: string  // 交代申請の場合
```

13. レスポンシブ対応

## 9. 非機能要件

|------|------|
| 初期表示速度 | 3秒以内 (LCP) |
| 同時利用 | 最大50ユーザー |
| データ保持 | 過去3年分 |

---

## 10. 考慮事項・リスク

- **シフト自動生成:** Phase 1 では手動作成に絞る。Phase 2 以降で自動生成ロジックを追加検討
- **給与計算連携:** 本システムでは集計のみ。給与システムとの連携はCSV出力で対応
- **個人情報保護:** 職員情報はFirestoreのセキュリティルールで厳密に保護する
- **オフライン対応:** Firestore のオフラインキャッシュで最低限の閲覧は可能
